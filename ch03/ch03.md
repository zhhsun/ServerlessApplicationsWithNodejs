# 第三章 我们Promise()，异步工作是很简单的

本章包括

- 使用Claudia处理异步操作

- JavaScript promises的基础

- 从Claudia和AWS Lambda连接到DynamoDB

在上一章中，您创建了一个用于处理披萨信息和订单的简单API。 您还了解到与传统的Node.js服务器不同，AWS Lambda状态在后续调用中丢失。 因此，需要数据库或外部服务来存储Aunt Maria的披萨订单或您想要保留的任何其他数据。

当Node.js异步执行时，您将首先了解无服务器是如何影响异步通信的：它如何与Claudia一起工作，更重要的是，什么是推荐的开发无服务器应用程序的方法。 当您掌握这些概念时，您将看到将AWS Lambda连接到外部服务是多么容易，您将学习如何使用它来使用AWS DynamoDB存储您的披萨订单。

因为我们的大脑不擅长异步阅读，书籍是以同步方式编写的，所以让我们一步一步走。
